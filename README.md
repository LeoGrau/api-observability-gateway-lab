# API Observability Gateway Lab

Experimental environment for evaluating **API Gateway and observability strategies** for HTTP-based microservices.

This project explores how gateway-based architectures and monitoring tools can provide visibility into **HTTP traffic, request metadata, payload inspection, and application performance**.

The environment simulates a microservice ecosystem running in **Docker containers**, fronted by an **API Gateway** capable of inspecting requests and forwarding traffic to backend services.

---

## Objectives

The goal of this repository is to evaluate solutions that allow:

- Inspection of **HTTP requests and responses**
- Visibility into **headers and request bodies**
- Monitoring of **API performance and latency**
- Centralized **traffic observability**
- Gateway-based routing for microservices
- Integration with **APM and tracing tools**

This setup aims to replicate capabilities similar to platforms such as:

- Azure API Management
- Application Insights
- Enterprise API observability stacks
